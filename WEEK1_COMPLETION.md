## ✨ 最佳实践总结

### 设计Prompt时
```
1. 先定义清晰的角色
   "你是一个[角色]"

2. 明确表达目的
   "你的目标是..."

3. 列举能力范围
   "你可以做..." / "你不能做..."

4. 选择合适的框架
   通常选择ReAct

5. 定义工具集合
   "可用工具有..."

6. 添加具体示例
   "Input:... Output:..."

7. 明确输出格式
   "Final Answer应该包括..."
```

### 评估Prompt时
```
使用6维度框架：
1. Clarity (清晰度)
2. Completeness (完整性)
3. Safety (安全性)
4. Examples (示例)
5. Actionable (可操作性)
6. Structure (结构性)

打分方法：
- 每个维度0-10分
- 计算总分（最高60分）
- 识别最弱的维度
- 针对性改进
```

### 优化Prompt时
```
1. 基于评估结果识别问题
2. 优先改进最弱维度
3. 一次改进一个问题
4. 重新评估并验证改进
5. 如果有改进就继续，否则停止
```
